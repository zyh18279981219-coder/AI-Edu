# 学生孪生对接字段说明

## 1. 文档目的
本文档用于说明学生数字孪生当前：

- 采集了哪些原始数据
- 生成了哪些画像结果
- 各字段中文名、参数名、含义和计算逻辑
- 能向课程孪生、教师孪生输出什么信息

当前主要服务入口为：

- Python 服务入口：`DigitalTwinModule.student_twin_service.StudentTwinService.build_summary(profile, trend)`
- 后端封装入口：`DigitalTwinModule.digital_twin_api.get_student_profile_summary(username)`

## 2. 当前采集的原始数据

| 中文名 | 参数名 | 含义 | 当前来源 |
|---|---|---|---|
| 用户名 | `username` | 学生唯一标识 | `data/Users/student.json` |
| 最后更新时间 | `last_updated` | 学生孪生画像最近一次刷新时间 | 存储层自动写入 |
| 知识点编号 | `node_id` | 某个知识点的唯一标识 | 课程树 / 知识图谱 |
| 知识点路径 | `node_path` | 该知识点在课程体系中的层级路径 | `CourseTree` 解析 |
| 测验成绩 | `quiz_score` | 学生在该知识点上的测验得分 | 测验模块回写 |
| 学习进度 | `progress` | 学生在该知识点上的完成度 | 学习进度数据 |
| 学习时长（分钟） | `study_duration_minutes` | 学生在该知识点上的累计学习时长 | 会话数据 |
| 互动次数 | `llm_interaction_count` | 学生围绕该知识点与 AI 或系统互动的次数 | LLM 日志 |
| 知识点掌握度 | `mastery_score` | 知识点综合掌握评分 | 系统计算生成 |

这些原始数据最终都会写入学生画像中的 `knowledge_nodes`。

## 4. 知识点掌握度计算逻辑

当前由 `DigitalTwinModule/score_calculator.py` 负责计算。

### 4.1 计算公式

```text
mastery_score
= 0.40 × 测验成绩 quiz_score
+ 0.30 × 学习进度 progress
+ 0.20 × min(互动次数 / 10, 1) × 100
+ 0.10 × min(学习时长分钟数 / 30, 1) × 100
```

### 4.2 规则补充

- 如果 `quiz_score` 缺失，则按 `0` 处理
- 如果 `quiz_score >= 100` 且 `progress >= 100`，则掌握度最低不低于 `90`
- 结果范围限制在 `0 ~ 100`

---

## 5. 整体掌握度计算逻辑

```text
overall_mastery = knowledge_nodes 中所有 mastery_score 的平均值
```

如果当前没有任何知识点数据，则返回 `0.0`。

---

## 6. 技术分层判定逻辑

当前由 `DigitalTwinModule/student_twin_service.py` 判定。

| 判定区间 | 输出层级 | 说明 |
|---|---|---|
| `overall_mastery < 40` | 基础薄弱 | 核心知识掌握较弱，需要优先补基础 |
| `40 <= overall_mastery < 60` | 基础建立中 | 已形成部分基础，但短板明显 |
| `60 <= overall_mastery < 80` 且薄弱点 `<= 4` | 能力成型 | 核心能力正在成型 |
| `60 <= overall_mastery < 80` 且薄弱点 `> 4` | 基础建立中 | 整体中等，但薄弱点仍较多 |
| `overall_mastery >= 80` | 进阶提升 | 整体掌握较好，可进入高阶训练 |

## 7. 能力雷达图字段与计算逻辑

当前雷达图包括 5 个维度：

| 能力名称 | 参数来源 | 含义 | 当前计算逻辑 |
|---|---|---|---|
| 知识掌握 | `overall_mastery` | 对课程知识点的整体理解和掌握程度 | `知识掌握 = overall_mastery` |
| 学习投入 | `llm_interaction_count`、`study_duration_minutes` | 学习中的投入程度和主动参与情况 | `学习投入 = 0.6 × interaction_avg + 0.4 × duration_avg` |
| 实践能力 | `progress`、`quiz_score`、投入综合 | 将学习推进到结果表现上的能力 | `实践能力 = 0.4 × progress_avg + 0.35 × quiz_avg + 0.25 × engagement` |
| 学习稳定性 | `trend.points[].overall_mastery` | 学习状态是否持续稳定 | `学习稳定性 = max(0, 100 - min(volatility × 4, 100))` |
| 测验表现 | `quiz_score`、`progress` | 在测验和阶段性反馈中的表现 | `测验表现 = max(quiz_avg, progress_avg × 0.8)` |

### 7.1 公式中的辅助量说明

| 英文参数 | 中文含义 | 说明 |
|---|---|---|
| `interaction_avg` | 平均互动投入分 | 每个知识点的互动次数，按 `10 次封顶` 归一化到 `0~100` 后再取平均 |
| `duration_avg` | 平均学习时长投入分 | 每个知识点的学习时长，按 `30 分钟封顶` 归一化到 `0~100` 后再取平均 |
| `progress_avg` | 平均学习进度 | 所有知识点 `progress` 的平均值 |
| `quiz_avg` | 平均测验成绩 | 所有知识点 `quiz_score` 的平均值 |
| `engagement` | 学习投入能力分 | 即上面的“学习投入”结果 |
| `volatility` | 学习波动值 | 相邻趋势点掌握度差值绝对值的平均值 |

## 8. 薄弱知识点输出逻辑

当前薄弱知识点来自：

- `knowledge_nodes` 中 `mastery_score < 60` 的知识点
- 按掌握度从低到高排序
- 如果节点本身没有 `node_path`，则通过 `CourseTree.resolve_node_path(node_id)` 补路径

返回字段如下：

| 中文名 | 参数名 | 含义 |
|---|---|---|
| 薄弱知识点名称 | `weak_nodes[].node_id` | 当前掌握较弱的知识点 |
| 薄弱知识点路径 | `weak_nodes[].node_path` | 该知识点所在课程路径 |
| 掌握度 | `weak_nodes[].mastery_score` | 当前掌握水平 |
| 学习进度 | `weak_nodes[].progress` | 当前学习推进情况 |
| 测验成绩 | `weak_nodes[].quiz_score` | 当前测验表现 |

---

## 9. 学习风险预警逻辑

| 风险类型 | 参数名 | 判断依据 | 当前规则 |
|---|---|---|---|
| 知识薄弱风险 | `knowledge_gap` | `overall_mastery` | `< 45` 高风险，`45~60` 中风险 |
| 进度滞后风险 | `progress_lag` | `average_progress` | `< 50` 高风险，`50~70` 中风险 |
| 学习投入风险 | `engagement_low` | `engagement` | `< 45` 中风险 |
| 趋势下滑风险 | `trend_down` | `trend_status` | `downward` 触发中风险 |
| 薄弱点集中风险 | `weak_nodes_cluster` | 薄弱点数量 | `>= 5` 触发中风险 |

风险字段格式：

| 中文名 | 参数名 | 含义 |
|---|---|---|
| 风险编码 | `risk_alerts[].code` | 风险内部标识 |
| 风险等级 | `risk_alerts[].level` | `high / medium / low` |
| 风险标题 | `risk_alerts[].title` | 风险名称 |
| 风险说明 | `risk_alerts[].detail` | 风险解释和建议 |

## 10. 趋势分析字段

| 中文名 | 参数名 | 含义 |
|---|---|---|
| 趋势状态 | `trend.trend_status` | `upward / stable / downward` |
| 趋势变化值 | `trend.change` | 一段时间内整体掌握度变化值 |
| 趋势说明 | `trend.summary` | 趋势文字说明 |
| 趋势点列表 | `trend.points` | 每日趋势数据点 |
| 趋势日期 | `trend.points[].date` | 日期 |
| 当日整体掌握度 | `trend.points[].overall_mastery` | 当天画像快照值 |

趋势判定逻辑：

- 变化值 `>= 5`：上升
- 变化值 `<= -5`：下降
- 其他：稳定

| 函数 | 所属模块 | 输入 | 输出 | 作用 |
|---|---|---|---|---|
| `collect_all(username)` | `DataCollector` | 用户名 | 无 | 一次性采集学习进度、互动、会话并刷新画像 |

## 15. 总结

当前学生孪生已经能够基于以下数据生成画像：

- 学习进度
- 测验成绩
- 学习时长
- AI 互动次数
- 趋势变化

并输出以下核心结果：

- 整体掌握度
- 技术分层
- 能力雷达图
- 薄弱知识点
- 学习风险预警
- 趋势分析
- 统计汇总
- 向课程孪生 / 教师孪生传递的中间结果
