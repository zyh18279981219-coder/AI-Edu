# 当前数据库表说明

数据库文件位置：

- `D:\develop\pythonFile\AI-Education2\data\app.db`

当前数据库使用 SQLite，本项目目前主要包含以下 8 张业务表。

## 1. `users`

用途：

- 存储系统用户信息
- 包含学生、教师、管理员账号

主要字段：

- `user_type`：用户类型，student / teacher / admin
- `username`：用户名
- `password`：密码
- `display_name`：显示名称
- `teacher`：所属教师
- `email`：邮箱
- `payload_json`：完整用户 JSON
- `updated_at`：更新时间

建表语句：

```sql
CREATE TABLE users (
    user_type TEXT NOT NULL,
    username TEXT NOT NULL,
    password TEXT,
    display_name TEXT,
    teacher TEXT,
    email TEXT,
    payload_json TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    PRIMARY KEY (user_type, username)
)
```

## 2. `teacher_student_links`

用途：

- 存储教师与学生的对应关系

主要字段：

- `teacher_username`：教师用户名
- `student_username`：学生用户名
- `updated_at`：更新时间

建表语句：

```sql
CREATE TABLE teacher_student_links (
    teacher_username TEXT NOT NULL,
    student_username TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    PRIMARY KEY (teacher_username, student_username)
)
```

## 3. `twin_profiles`

用途：

- 存储学生数字孪生当前画像

主要字段：

- `username`：学生用户名
- `last_updated`：画像最近更新时间
- `overall_mastery`：整体掌握度
- `payload_json`：完整孪生画像 JSON
- `updated_at`：数据库更新时间

建表语句：

```sql
CREATE TABLE twin_profiles (
    username TEXT PRIMARY KEY,
    last_updated TEXT,
    overall_mastery REAL DEFAULT 0,
    payload_json TEXT NOT NULL,
    updated_at TEXT NOT NULL
)
```

## 4. `twin_history`

用途：

- 存储学生数字孪生历史快照
- 用于趋势分析

主要字段：

- `username`：学生用户名
- `snapshot_date`：快照日期
- `overall_mastery`：当日整体掌握度
- `payload_json`：完整历史快照 JSON
- `updated_at`：数据库更新时间

建表语句：

```sql
CREATE TABLE twin_history (
    username TEXT NOT NULL,
    snapshot_date TEXT NOT NULL,
    overall_mastery REAL DEFAULT 0,
    payload_json TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    PRIMARY KEY (username, snapshot_date)
)
```

## 5. `sessions`

用途：

- 存储登录会话
- 保存当前用户、当前 PDF、当前节点等状态

主要字段：

- `session_id`：会话 ID
- `username`：用户名
- `user_type`：用户类型
- `created_at`：创建时间
- `last_accessed`：最后访问时间
- `current_pdf_path`：当前选中的 PDF 路径
- `current_node`：当前节点
- `payload_json`：完整 session JSON
- `updated_at`：更新时间

建表语句：

```sql
CREATE TABLE sessions (
    session_id TEXT PRIMARY KEY,
    username TEXT NOT NULL,
    user_type TEXT NOT NULL,
    created_at TEXT,
    last_accessed TEXT,
    current_pdf_path TEXT,
    current_node TEXT,
    payload_json TEXT NOT NULL,
    updated_at TEXT NOT NULL
)
```

## 6. `user_states`

用途：

- 存储用户页面状态、学习状态等轻量状态数据

主要字段：

- `username`：用户名
- `payload_json`：状态 JSON
- `updated_at`：更新时间

建表语句：

```sql
CREATE TABLE user_states (
    username TEXT PRIMARY KEY,
    payload_json TEXT NOT NULL,
    updated_at TEXT NOT NULL
)
```

## 7. `llm_logs`

用途：

- 存储大模型调用日志

主要字段：

- `id`：自增主键
- `timestamp`：调用时间
- `username`：用户
- `module`：调用模块
- `model`：模型名
- `payload_json`：完整日志 JSON

建表语句：

```sql
CREATE TABLE llm_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TEXT,
    username TEXT,
    module TEXT,
    model TEXT,
    payload_json TEXT NOT NULL
)
```

## 8. `learning_plans`

用途：

- 存储学习计划
- 也存储个性化学习路径结果

主要字段：

- `username`：用户名
- `filename`：文件名标识
- `plan_path`：原始路径字段
- `category`：类别，常见为 `global`、`user`、`path`
- `payload_json`：完整计划 JSON
- `updated_at`：更新时间

建表语句：

```sql
CREATE TABLE learning_plans (
    username TEXT NOT NULL,
    filename TEXT NOT NULL,
    plan_path TEXT,
    category TEXT,
    payload_json TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    PRIMARY KEY (username, filename)
)
```

## 当前结论

当前系统核心数据已经主要落到 SQLite 中，重点围绕以下几类：

- 用户与登录：`users`、`sessions`、`user_states`
- 学生数字孪生：`twin_profiles`、`twin_history`
- 学习计划与学习路径：`learning_plans`
- 教师学生关系：`teacher_student_links`
- 大模型日志：`llm_logs`

如果后续要继续做数据库公开设计或项目对接，建议优先基于这 8 张表继续扩展。
