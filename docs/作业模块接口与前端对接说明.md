# 作业模块接口与前端对接说明

## 1. 模块目标

本模块提供以下能力：

1. 教师发布作业（主观题 / 代码实践）
2. 学生提交作业
3. 教师 AI 辅助批改 + 人工终审
4. 教师 AI 辅助出题
5. 作业数据默认直写 SQLite（兼容旧 JSON 迁移）

后端路由前缀：`/api/homework`

## 2. 鉴权与角色

所有接口均依赖现有登录态 Cookie（`session_id`）。

1. 教师可用：发布作业、查看提交、AI批改、终审、AI出题、导出
2. 学生可用：查看作业、提交作业、查看自己的提交
3. 管理员可用：导出到 SQLite

## 3. 主要接口

### 3.1 发布作业

- 方法：`POST /api/homework/assignments`
- 角色：教师

请求示例：

```json
{
  "title": "排序算法综合作业",
  "description": "包含分析与编码",
  "assignment_type": "code_practice",
  "class_name": "软件工程1班",
  "due_at": "2026-04-20T23:59:00",
  "rubric": "正确性40，复杂度20，规范20，说明20",
  "questions": [
    {
      "title": "快速排序实现",
      "prompt": "请实现快速排序并分析复杂度",
      "reference_answer": "可选",
      "rubric": "可选",
      "test_cases": []
    }
  ]
}
```

### 3.2 获取作业列表

- 方法：`GET /api/homework/assignments?only_mine=true`
- 角色：已登录用户（教师建议传 only_mine=true）

### 3.3 学生提交作业

- 方法：`POST /api/homework/assignments/{assignment_id}/submissions`
- 角色：学生

请求示例：

```json
{
  "answers": [
    {
      "question_index": 0,
      "answer": "我的答案内容"
    }
  ]
}
```

### 3.4 查看某作业提交

- 方法：`GET /api/homework/assignments/{assignment_id}/submissions`
- 角色：发布该作业的教师

### 3.5 AI 辅助批改

- 方法：`POST /api/homework/submissions/{submission_id}/ai-grade`
- 角色：发布该作业的教师

请求示例：

```json
{
  "force_regenerate": true
}
```

### 3.6 教师终审

- 方法：`POST /api/homework/submissions/{submission_id}/final-grade`
- 角色：发布该作业的教师

请求示例：

```json
{
  "teacher_score": 88,
  "teacher_comment": "教师终审通过"
}
```

### 3.7 AI 辅助出题

- 方法：`POST /api/homework/ai/generate-questions`
- 角色：教师

请求示例：

```json
{
  "topic": "二叉树遍历",
  "assignment_type": "code_practice",
  "count": 3,
  "difficulty": "medium",
  "language": "zh",
  "extra_requirements": "包含复杂度分析"
}
```

### 3.8 兼容迁移旧 JSON 到 SQLite

- 方法：`POST /api/homework/export/sqlite`
- 角色：教师 / 管理员

说明：当前作业数据已经默认写入 SQLite。本接口用于把历史 `data/homework/homework_store.json` 迁移进 SQLite。

返回示例：

```json
{
  "success": true,
  "operator": "teawangqiyu",
  "result": {
    "success": true,
    "assignments_exported": 12,
    "submissions_exported": 87,
    "sqlite_db_path": "data/app.db",
    "exported_at": "2026-04-13T21:00:00"
  }
}
```

## 4. SQLite 主存储结构

导出后将落到以下两张表（自动创建）：

1. `homework_assignments`
2. `homework_submissions`

其中题目与答案分别存储在：

1. `questions_json`
2. `answers_json`

方便后续逐步迁移到规范化表结构。

## 5. 命令行迁移旧 JSON

可直接运行：

```bash
/Users/shadowfall/PycharmProjects/AI-Edu/.venv/bin/python tools/export_db_seed.py --mode export-homework
```

可选参数：

1. `--homework-store`（默认 `data/homework/homework_store.json`）
2. `--db`（默认 `data/app.db`）

## 6. 前端接入点

教师前端页面与 API 封装：

1. 页面：`frontend-vue/src/views/teacher/TeacherHomeworkView.vue`
2. API：`frontend-vue/src/api/homework.ts`
3. 类型：`frontend-vue/src/types/homework.ts`
4. 路由：`/teacher/homework`

## 7. 低耦合说明

本模块采用独立目录 `HomeworkModule`，仅复用项目公共能力：

1. 登录态会话管理
2. 用户角色校验
3. LLM 日志记录

不修改原有教师看板、测验、学习计划、数字孪生等模块内部逻辑，便于多人并行开发。
