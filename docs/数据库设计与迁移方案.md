# 数据库设计与迁移方案（实体化落地版）

> 同步日期：2026-04-14

## 1. 当前完成状态

### 1.1 用户主键升级
- `users` 物理主键：`user_id`
- 保留：`UNIQUE(user_type, username)`
- 登录主标识：`login_id/user_id`

### 1.2 学生孪生拆分
- `twin_profiles`：主信息聚合
- `twin_profile_nodes`：节点明细
- `twin_history`：趋势历史

### 1.3 课程/资料/测试实体化
- `courses`（`course_id`）
- `course_nodes`（`course_id + node_id`）
- `resources`（`resource_id`）
- `quiz_attempts`（`attempt_id`）

## 2. 当前读写策略

- 读取：课程相关接口实体优先（`knowledge-graph/learning-nodes/node-resources`）
- 写入：`upload/delete-resource/quiz-complete` 已切实体主写
- 旧 JSON：运行时不再作为主写路径，仅保留“实体为空时一次性导入兜底”。

## 3. 已落地索引策略（关键）

- 用户：`idx_users_user_id`、`idx_users_login_id`
- 课程节点：`idx_course_nodes_course`
- 资源：`idx_resources_course_node`、`idx_resources_path`
- 测试：`idx_quiz_attempts_user_id`、`idx_quiz_attempts_course_id`、`idx_quiz_attempts_created_at`

## 4. 风险与控制

1. 端口污染风险：旧进程占用 8000/8010 导致“改了未生效”错觉。
2. 兼容风险：极少数仍依赖历史 JSON 路径的辅助逻辑需持续巡检。
3. 数据一致性风险：实体与历史文件并存阶段，严格以实体为准。

## 5. 验收方式

```powershell
python scripts\backend_api_selftest.py --base-url http://127.0.0.1:8010
```

验收重点：
- 登录链路（`login_id/user_id`）
- 学生课程孪生
- 教师外部同步
- 实体表写入（课程/资源/测试）

## 6. 后续建议

1. 将“实体为空兜底导入”迁移到独立迁移命令，不在运行时触发。
2. 前端逐步消费实体 ID（`course_id/resource_id/attempt_id`）。
3. 新增实体一致性巡检脚本（每天或每次部署后跑）。
