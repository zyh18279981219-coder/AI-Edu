# 学生数字孪生设计与对接文档

# 一、有哪些功能

## 1.1 学生画像生成

- 输入：学生标识、课程标识、学习过程数据、测验数据、节点掌握数据
- 输出：课程维度学生画像（JSON）
- 核心指标：
  - `overall_mastery`：整体掌握度
  - `technical_level`：能力分层（基础薄弱/基础建立中/能力成型/进阶提升）
  - `radar`：知识掌握、学习投入、实践能力、学习稳定性、测验表现
  - `risk_alerts`：风险告警（知识薄弱、进度滞后、投入不足、趋势下滑等）
  - `trend`：趋势序列（按日/按采样点）
  
  

## 1.2 学生-课程画像查询能力

- API：`POST /api/digital-twin/student-course-profile`
- 入参：`student_id`、`course_id`
- 出参：该生该课完整画像 + 节点匹配结果 + 对其他孪生的输出字段

## 1.3 对其他孪生输出能力（跨模块协同）

- 输出给课程孪生：`outputs.for_course_twin`
  - 课程总体掌握度、风险等级、薄弱节点摘要
- 输出给教师孪生：`outputs.for_teacher_twin`
  - 技术等级、风险列表、趋势状态、薄弱节点摘要

## 1.4 双调用模式

- HTTP 模式（前后端/跨服务）  
  调用数字孪生 API 获取 JSON。
- 函数模式（后端内部）  
  `DigitalTwinModule.student_course_profile_service.get_student_course_profile(student_id, course_id)`  
  返回 Python `dict`（可直接序列化为 JSON）。

## 1.5 数据更新与回写

- 测验完成后更新节点状态与测验记录（`quiz_attempts`）
- 定时/触发式采集后更新 `twin_profiles` 与 `twin_profile_nodes`
- 画像读取优先实体表，保证可查询、可审计、可扩展



# 二、数据库表设计（已有）

可开发新表或者在已有表中添加新的字段

## 1. users（用户主数据）

| 字段                                                         | 类型    | 约束/索引                                        | 含义                                   |
| ------------------------------------------------------------ | ------- | ------------------------------------------------ | -------------------------------------- |
| user_id                                                      | INTEGER | PRIMARY KEY，唯一索引 `idx_users_user_id`        | 用户唯一数字 ID（物理主键 + 主查询键） |
| login_id                                                     | TEXT    | UNIQUE 索引 `idx_users_login_id`                 | 登录账号（主登录键）                   |
| user_type                                                    | TEXT    | NOT NULL，索引 `idx_users_type`                  | 用户类型：`student/teacher/admin`      |
| username                                                     | TEXT    | NOT NULL，唯一约束 `UNIQUE(user_type, username)` | 历史用户名（兼容字段，按类型唯一）     |
| password                                                     | TEXT    |                                                  | 登录密码                               |
| display_name                                                 | TEXT    |                                                  | 展示名称                               |
| teacher                                                      | TEXT    |                                                  | 学生关联教师（历史业务字段）           |
| email                                                        | TEXT    |                                                  | 邮箱                                   |
| payload_json                                                 | TEXT    | NOT NULL                                         | 用户完整 JSON 快照                     |
| updated_at                                                   | TEXT    | NOT NULL                                         | 记录更新时间（ISO 字符串）             |
| 索引：`idx_users_type`、`idx_users_user_id`、`idx_users_login_id` |         |                                                  |                                        |

## 2. teacher_student_links（师生关系）

| 字段             | 类型    | 约束/索引                        | 含义                 |
| ---------------- | ------- | -------------------------------- | -------------------- |
| teacher_username | TEXT    | NOT NULL，主键组成               | 教师用户名（历史键） |
| student_username | TEXT    | NOT NULL，主键组成               | 学生用户名（历史键） |
