# AI-Education 系统接口说明

本文档说明当前项目对外提供的主要 HTTP 接口，供前端联调、上线交接与同学使用时排查问题。

## 1. 认证相关

### `POST /api/auth/login`

- 说明：统一登录接口
- 请求体：

```json
{
  "username": "zyh",
  "password": "123456",
  "user_type": "student"
}
```

- 返回：登录结果与当前用户信息

### `POST /api/logout`

- 说明：退出登录

### `GET /api/current-user`

- 说明：获取当前登录用户信息

## 2. 个人资料相关

### `POST /api/update-profile`

- 说明：更新学生个人资料

### `POST /api/change-password`

- 说明：修改学生密码

## 3. 课程学习相关

### `GET /api/knowledge-graph`

- 说明：获取课程知识树

### `GET /api/graph-visualization`

- 说明：首页知识图谱可视化数据

### `GET /api/learning-nodes`

- 说明：获取学习节点列表

### `POST /api/node/resources`

- 说明：获取节点资源

### `POST /api/pdf/select`

- 说明：选择当前学习 PDF

### `GET /api/pdf/{path}`

- 说明：读取 PDF 文件

### `GET /api/learning-progress`

- 说明：获取学习进度统计

## 4. AI 助教与总结

### `POST /api/chat`

- 说明：课程问答

### `POST /api/summary`

- 说明：生成当前主题总结

### `GET /api/health/llm`

- 说明：检测大模型服务是否可用

## 5. 测验相关

### `POST /api/quiz/start`

- 说明：开始测验

### `POST /api/quiz/answer`

- 说明：提交单题答案

### `POST /api/quiz/summary`

- 说明：生成测验总结

### `POST /api/quiz/complete`

- 说明：测验完成后回写课程节点完成状态

## 6. 学习计划相关

### `POST /api/learning-plan`

- 说明：根据目标生成学习计划

### `POST /api/learning-plan/from-quiz`

- 说明：根据测验结果生成学习计划

### `GET /api/learning-plans`

- 说明：获取当前用户学习计划列表
- 当前返回逻辑：
  - 仅返回 `global`、`user` 两类学习计划
  - 已过滤学习路径 `path` 类型数据

## 7. 学生数字孪生相关

接口前缀：`/api/digital-twin`

### `POST /api/digital-twin/collect/{username}`

- 说明：采集并刷新学生孪生数据

### `GET /api/digital-twin/profile/{username}`

- 说明：获取原始孪生画像

### `GET /api/digital-twin/student-profile/{username}`

- 说明：获取前端展示用的学生孪生摘要

### `POST /api/digital-twin/quiz-score`

- 说明：同步测验得分到学生孪生

### `POST /api/digital-twin/path/generate/{username}`

- 说明：生成个性化学习路径

### `GET /api/digital-twin/path/{username}/current`

- 说明：获取当前最新学习路径

### `PATCH /api/digital-twin/path/{username}/node/{node_id}`

- 说明：更新某个薄弱节点掌握度后，联动刷新路径

## 8. 教师端相关

接口前缀：`/api/dashboard`

### `GET /api/dashboard/class-overview`

- 说明：教师班级概览

### `GET /api/dashboard/student/{username}`

- 说明：查看某个学生详情

### `GET /api/dashboard/student/{username}/trend`

- 说明：查看某个学生趋势

### `GET /api/dashboard/node/{node_id}/ranking`

- 说明：查看知识点学生排名

### `GET /api/students`

- 说明：教师端获取学生列表

### `GET /api/teachers`

- 说明：管理员或系统侧获取教师列表

### `GET /api/heatmap`

- 说明：教师端课程热力图数据

## 9. 行业情报相关

接口前缀：`/api/industry-intelligence`

### `GET /api/industry-intelligence/status`

- 说明：获取行业情报基础配置状态

### `GET /api/industry-intelligence/current`

- 说明：获取当前任务状态

### `GET /api/industry-intelligence/tasks/{task_id}`

- 说明：按任务 ID 查询状态

### `POST /api/industry-intelligence/tasks/{task_id}/cancel`

- 说明：终止任务

### `POST /api/industry-intelligence/analyze`

- 说明：启动行业情报采集与分析

### `POST /api/industry-intelligence/reanalyze`

- 说明：对已有岗位重新分析

## 10. 管理端相关

### `GET /api/llm-logs`

- 说明：获取 LLM 调用日志

### `POST /api/register`

- 说明：注册学生或教师

## 11. 文件上传与 OCR

### `POST /api/upload`

- 说明：上传课程资源并写入知识树

### `POST /api/delete-resource`

- 说明：删除节点资源

### `POST /api/ocr/extract`

- 说明：图片 OCR 文本提取

## 12. 说明

- 当前系统对外是 HTTP 接口
- 系统内部模块之间主要仍通过 Python 函数和类调用完成业务处理
- 数据持久化当前以本地 SQLite 为主，数据库文件为 `data/app.db`
