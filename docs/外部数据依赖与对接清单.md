# 外部数据依赖与对接清单

## 1. 目标
本文档说明：

- 当前系统还需要从其他同事或其他模块拿到什么数据
- 这些数据会被哪个模块使用
- 用于生成什么结果

---

## 2. 学生孪生需要的外部数据

| 需要的数据 | 当前来源 | 当前用途 | 是否已接通 |
|---|---|---|---|
| 学习进度数据 | 学习模块 | 计算 `progress`、整体掌握度、实践能力 | 已接通 |
| 测验成绩数据 | 测验模块 | 计算 `quiz_score`、掌握度、测验表现 | 已接通 |
| LLM 互动日志 | AI 助教 / 总结 / 智能体模块 | 计算 `llm_interaction_count`、学习投入 | 已接通 |
| 会话时长 / 当前学习节点 | 会话管理模块 | 计算 `study_duration_minutes`、定位当前学习状态 | 已接通 |
| 课程树 / 知识图谱结构 | 课程模块 / Neo4j | 解析 `node_path`、补路径、关联学习路径 | 已接通 |
| 更细粒度实践行为 | 练习模块 / 编码模块 | 丰富实践能力、风险识别 | 未完全接通 |
| 教师干预记录 | 教师模块 | 反哺学生孪生的风险解释和策略调整 | 未接通 |

## 6. 当前最需要其他同事补齐的内容

如果按优先级看，最建议尽快向其他同事确认这几块：

1. 课程模块是否能稳定提供知识点标准 ID、路径和依赖关系
2. 教师模块是否能提供教师干预记录、课堂反馈、班级分析输入
3. 练习 / 编码模块是否能提供更细粒度实践行为数据
4. 资源模块是否能提供资源点击率、完成率、使用效果数据

---

## 7. 对接建议

建议项目组对接时，不要只说“给我数据”，而是按下面方式约定：

- 数据名称
- 字段清单
- 更新频率
- 来源模块
- 负责同事
- 提供方式：Python 函数 / 文件 / 表 / 事件

这样后续才能真正把三大数字孪生串起来。
