

# 一、有哪些功能


## 1.1 深度能力画像模块


- 输入：学生在各知识点上的原始作答结果、每道题的得分与用时、题目所属知识点、历史测验最高分记录、最近一段时间的连续作答稳定性数据。
- 输出：能力白皮书（自然语言报告 + 结构化结果）。
- API 配置：
	- 接口：`POST /api/diagnosis/persona-profile`
	- 入参：学生标识、课程标识（可选时间窗口）
	- 出参：能力白皮书、语义化能力标签、能力结构说明
- 深度体现：实现从“分数值”到“人设画像”的跨越。


## 1.2 溯源归因诊断模块


- 输入：错题明细、每次作答的题目编号与知识点归属、知识图谱中的前驱后继关系、与智能助手的交互日志、做题开始与结束时间、历史测验分数序列。
- 输出：归因链路图（结构化结果）+ 根因解释报告（自然语言）。
- API 配置：
	- 接口：`POST /api/diagnosis/root-cause-trace`
	- 入参：学生标识、课程标识（可选指定知识点或题目范围）
	- 出参：归因链路图、根因分类（知识遗忘/逻辑断层/迁移障碍）、诊断解释
- 深度体现：实现从“关联关系”到“因果逻辑”的跨越。


## 1.3 动态风险前瞻模块

- 输入：按时间排列的测验分数序列、学习时长原始记录、单次学习停留时长、连续多天的作答变化、知识点复习间隔、近期行为波动记录。
- 输出：风险演习报告（自然语言）+ 风险概率结果（结构化）。
- API 配置：
	- 接口：`POST /api/diagnosis/risk-forecast`
	- 入参：学生标识、课程标识（可选预测周期，如 3 天/7 天）
	- 出参：风险等级、失败概率、知识半衰期提示、干预时间建议
- 深度体现：实现从“静态评估”到“动态演化”的跨越。


## 1.4 群体洞察扩展

- 输入：按课程汇总的多名学生原始学习与测验数据，包括作答结果、题目得分、答题时长、知识点归属、交互日志、行为时间戳、课程节点结构与学生名单。
- 输出：群体诊断看板（可读报告 + 结构化结果）。
- API 配置：
	- 接口：`POST /api/diagnosis/group-insight`
	- 入参：课程标识、学生列表（可选，不传默认全班）、统计时间窗口
	- 出参：群体能力分布、薄弱节点热力、趋势分群、群体干预优先级





# 二、数据库表设计



## 2.1 新增表建议

### 1. diagnosis_reports（学生级诊断主表）

| 字段 | 类型 | 约束/索引 | 含义 |
| --- | --- | --- | --- |
| report_id | INTEGER | PRIMARY KEY AUTOINCREMENT | 诊断报告主键 |
| user_id | INTEGER | NOT NULL，索引 `idx_diag_reports_user` | 学生 ID |
| course_id | TEXT | NOT NULL，索引 `idx_diag_reports_course` | 课程 ID |
| report_date | TEXT | NOT NULL，索引 `idx_diag_reports_date` | 报告日期（YYYY-MM-DD） |
| persona_summary | TEXT | NOT NULL | 能力画像摘要 |
| root_cause_summary | TEXT | NOT NULL | 根因诊断摘要 |
| risk_summary | TEXT | NOT NULL | 风险前瞻摘要 |
| risk_level | TEXT |  | 风险等级（low/medium/high） |
| causal_chain_json | TEXT | NOT NULL | 归因链路与根因证据 |
| risk_forecast_json | TEXT | NOT NULL | 风险预测与干预建议 |
| payload_json | TEXT | NOT NULL | 完整结构化报告 |
| created_at | TEXT | NOT NULL | 创建时间 |
| updated_at | TEXT | NOT NULL | 更新时间 |

建议唯一约束：`UNIQUE (user_id, course_id, report_date)`

### 2. diagnosis_group_snapshots（群体诊断快照）

| 字段 | 类型 | 约束/索引 | 含义 |
| --- | --- | --- | --- |
| snapshot_id | INTEGER | PRIMARY KEY AUTOINCREMENT | 快照主键 |
| course_id | TEXT | NOT NULL，索引 `idx_group_course_id` | 课程 ID |
| snapshot_date | TEXT | NOT NULL，索引 `idx_group_snapshot_date` | 快照日期 |
| sample_size | INTEGER | NOT NULL | 样本人数 |
| mastery_distribution_json | TEXT | NOT NULL | 群体掌握度分布 |
| weak_nodes_json | TEXT | NOT NULL | 高频薄弱节点 |
| trend_clusters_json | TEXT | NOT NULL | 趋势分群结果 |
| risk_heatmap_json | TEXT | NOT NULL | 风险热力图 |
| teacher_summary | TEXT |  | 教师可读摘要 |
| created_at | TEXT | NOT NULL | 创建时间 |
| updated_at | TEXT | NOT NULL | 更新时间 |

建议唯一约束：`UNIQUE (course_id, snapshot_date)`

说明：
- 归因链路明细与风险预测明细不再单独建表，先统一沉淀到 `diagnosis_reports.causal_chain_json`、`diagnosis_reports.risk_forecast_json` 与 `payload_json`。
- 若后续数据量显著增长或需要高频回测，再拆分独立明细表。



# 三、需要补充的数据



## 3.1 建议补充的数据

- 题目级作答过程：每题作答时长、修改次数、是否多次反复作答。
- 交互细粒度日志：提问类型、提问轮次、追问深度、关键错误关键词。
- 代码评测或作业评测结果（含题目维度得分和错误类型）。
- 班级与分组映射关系（支持群体统计和分组对比）。
