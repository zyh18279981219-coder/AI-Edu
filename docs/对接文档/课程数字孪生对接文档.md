# 课程数字孪生设计与对接文档

> 本文档用于定义“知识图谱 + 课程数字孪生”子系统与学生数字孪生、教师数字孪生、个性化路径推荐、诊断智能体之间的输入输出、接口与数据表约定。  
> 当前设计基于以下事实：
> - 当前仓库已具备课程知识图谱抽取能力，现有输出可落到 `课程 -> 章节 -> 小节 -> 知识点`
> - 当前仓库已具备资源匹配能力，现有输出可落到 `小节 -> 候选资源 -> 匹配分数`
> - 项目整体要求课程数字孪生承担课程内容重构、资源关联、评价标准管理、课程演化与班级层分析职责

# 一、有哪些功能

## 1.1 课程画像生成

- 输入：课程基础信息、知识图谱数据、节点关系数据、资源匹配数据、学生数字孪生聚合结果、教师数字孪生反馈、评价标准配置、行业能力需求/课程目标
- 输出：课程维度课程画像（JSON）
- 课程画像由三层组成：
  - 结构层：课程图谱、章节/小节/知识点关系、先修/相关/包含关系、能力映射
  - 资源层：节点资源网络、资源难度、资源匹配度、资源覆盖缺口
  - 演化层：班级掌握热力、教学重点/风险点、评价标准、版本迭代历史
- 核心指标：
  - `graph_coverage_rate`：知识图谱覆盖率
  - `resource_coverage_rate`：资源覆盖率
  - `rubric_coverage_rate`：评价标准覆盖率
  - `overall_mastery`：课程整体掌握度（班级/群体维度）
  - `difficulty_distribution`：节点难度分布
  - `risk_nodes`：高风险节点列表
  - `hot_nodes`：高频访问/高价值节点列表
  - `iteration_version`：当前课程孪生版本
  - `evolution_signals`：来自学生、教师、行业反馈的演化信号

## 1.2 课程基础图谱接入能力

- API：`POST /api/digital-twin/course-ingest/knowledge-graph`
- 入参：`course_id`、`course_name`、`term_id`、`graph_payload`
- `graph_payload` 最小应兼容当前两类知识图谱产物：
  - 原始课程图谱：`course_name -> chapters[].chapter_name -> points[] -> relations[]`
  - 标准化课程图谱：`course_name -> chapters[].chapter_name -> sections[].section_name -> knowledge_points[] -> relations[]`
- 推荐统一后的标准写入结构如下：
  - `course_name`
  - `chapters[].chapter_name`
  - `chapters[].sections[].section_name`
  - `chapters[].sections[].knowledge_points[].name`
  - `chapters[].sections[].knowledge_points[].desc`
  - `relations[].from / to / type`
- 出参：`course_id`、`version_no`、`ingested_nodes`、`ingested_relations`、`status`
- 作用：将现有知识图谱结果固化为课程数字孪生的结构层数据，供课程画像、路径推荐、诊断智能体复用

## 1.3 课程资源网络接入能力

- API：`POST /api/digital-twin/course-ingest/resource-matches`
- 入参：`course_id`、`term_id`、`resource_payload`
- `resource_payload` 最小应支持当前资源匹配产物中的以下结构：
  - `chapter_name`
  - `section_name`
  - `queries`
  - `matches[].platform`
  - `matches[].url`
  - `matches[].title`
  - `matches[].snippet`
  - `matches[].matched_queries`
  - `matches[].score`
  - `matches[].final_score`
- 出参：`course_id`、`linked_nodes`、`linked_resources`、`status`
- 作用：把当前 `小节 -> 资源候选 -> 匹配得分` 的结果写入课程数字孪生资源层

## 1.4 来自学生与教师侧的动态更新能力

- API：`POST /api/digital-twin/course-ingest/student-aggregation`
- 入参：`course_id`、`term_id`、`class_id`、`student_node_metrics`
- `student_node_metrics` 最小字段建议：
  - `student_id`
  - `node_id`
  - `mastery_score`
  - `progress`
  - `quiz_score`
  - `study_duration_minutes`
  - `llm_interaction_count`
  - `risk_flags`
  - `updated_at`
- 作用：把学生数字孪生的节点级画像汇总到课程数字孪生，形成班级热力、掌握分布、风险节点

- API：`POST /api/digital-twin/course-ingest/teacher-feedback`
- 入参：`course_id`、`term_id`、`class_id`、`teacher_feedback`
- `teacher_feedback` 最小字段建议：
  - `focus_nodes`
  - `teaching_strategy`
  - `content_adjustments`
  - `intervention_summary`
  - `rubric_updates`
  - `updated_at`
- 作用：把教师数字孪生输出的教学策略、重难点判断、内容调整建议写回课程数字孪生

## 1.5 课程画像查询能力

- API：`POST /api/digital-twin/course-profile`
- 入参：
  - `course_id`
  - `term_id`
  - `class_id`
  - `include_nodes`
  - `include_resources`
  - `include_outputs`
- 出参：完整课程画像 + 节点热力结果 + 资源网络结果 + 对其他孪生/智能体的输出字段

## 1.6 课程节点与资源查询能力

- API：`POST /api/digital-twin/course-node-profile`
- 入参：`course_id`、`term_id`、`class_id`、`node_id`
- 出参：该节点的结构信息、先修关系、资源情况、掌握情况、风险情况、建议动作

- API：`POST /api/digital-twin/course-resource-map`
- 入参：`course_id`、`term_id`、`node_id`、`resource_role`、`top_k`
- 出参：节点资源列表、资源匹配分、资源难度、推荐理由、未覆盖情况

- API：`POST /api/digital-twin/course-rubric`
- 入参：`course_id`、`term_id`
- 出参：课程评价指标编码、权重、证据来源、评分方式

## 1.7 对其他孪生与智能体输出能力（跨模块协同）

- 输出给学生孪生：`outputs.for_student_twin`
  - 下一跳推荐节点
  - 先修缺口链路
  - 适配资源列表
  - 节点难度与建议节奏

- 输出给教师孪生：`outputs.for_teacher_twin`
  - 班级学习画像
  - 节点掌握热力图
  - 高风险节点/高风险群体摘要
  - 教学重点建议与内容调整建议

- 输出给路径推荐模块：`outputs.for_path_recommender`
  - 图谱拓扑
  - 先修关系
  - 节点重要度
  - 节点难度
  - 候选资源

- 输出给诊断智能体：`outputs.for_diagnosis_agent`
  - 症状对应节点
  - 先修缺口链
  - 证据节点
  - 易错知识点与可能误区

- 输出给课程自进化模块：`outputs.for_course_evolution`
  - 缺口资源节点
  - 低效果资源
  - 评价标准空洞点
  - 建议补全/重构节点

## 1.8 双调用模式

- HTTP 模式（前后端/跨服务）
  - 通过课程数字孪生 API 获取或更新 JSON

- 函数模式（后端内部）
  - `DigitalTwinModule.course_twin_ingest_service.ingest_knowledge_graph(course_id, payload)`
  - `DigitalTwinModule.course_twin_ingest_service.ingest_resource_matches(course_id, payload)`
  - `DigitalTwinModule.course_twin_ingest_service.ingest_student_aggregation(course_id, term_id, class_id, payload)`
  - `DigitalTwinModule.course_twin_ingest_service.ingest_teacher_feedback(course_id, term_id, class_id, payload)`
  - `DigitalTwinModule.course_twin_profile_service.get_course_profile(course_id, term_id="global", class_id="global")`
  - `DigitalTwinModule.course_twin_profile_service.get_course_node_profile(course_id, node_id, term_id="global", class_id="global")`

## 1.9 数据更新与回写

- 知识图谱构建完成后，更新 `courses`、`course_nodes`、`course_node_relations`
- 资源匹配完成后，更新 `resources` 与 `course_resource_links`
- 学生数字孪生定时/触发式更新后，聚合更新 `course_twin_profiles` 与 `course_twin_nodes`
- 教师数字孪生输出新的教学策略、评价标准或内容调整建议后，更新 `course_evaluation_rubrics` 与 `course_twin_profiles.payload_json`
- 手动发布版本或定时快照时，写入 `course_twin_history`
- 课程画像读取优先当前快照表，历史分析与审计读取 `course_twin_history`

## 1.10 职责边界

- 课程数字孪生负责课程结构、资源网络、评价标准、班级层聚合与课程演化
- 学生数字孪生负责学生个体画像、个体趋势、个体风险
- 教师数字孪生负责教师教学行为分析、教师反思与干预建议
- 课程数字孪生不替代学生/教师个体画像，但负责为它们提供课程上下文、节点关系和资源依据

# 三、数据库表设计（复用 + 建议新增）

## 3.1 可直接复用的已有表

| 表名 | 当前作用 | 在课程数字孪生中的用途 |
| --- | --- | --- |
| `courses` | 课程主数据与原始结构快照 | 保存课程基础信息与原始导入快照 |
| `course_nodes` | 课程节点信息 | 保存章节/小节/知识点等树状节点 |
| `resources` | 课程资料实体 | 保存视频、文档、案例、题目等资源实体 |
| `quiz_attempts` | 测试记录 | 用于课程节点掌握度聚合与班级层分析 |
| `twin_profile_nodes` | 学生节点级画像 | 聚合生成课程节点热力、风险节点、整体掌握度 |
| `twin_profiles` | 学生整体画像 | 用于计算课程整体掌握度、群体能力分布 |
| `learning_plans` | 学习计划 | 可回写课程侧推荐路径或节点序列 |

> 说明：课程数字孪生不建议重复建设 `courses`、`course_nodes`、`resources` 三张实体表，应在其上增加关系层、画像层、历史层。

## 3.2 建议新增表

### 1. course_twin_profiles（课程孪生主表）

> `class_id` 统一约定：  
> - 课程全局视图用 `global`  
> - 班级视图用真实 `class_id`  
> 若当前系统暂无 `class_id`，可暂时用 `teacher_user_id + course_id` 作为代理键。

| 字段 | 类型 | 约束/索引 | 含义 |
| --- | --- | --- | --- |
| course_id | TEXT | NOT NULL，主键组成，索引 `idx_course_twin_profiles_course` | 课程 ID |
| term_id | TEXT | NOT NULL，主键组成 | 学期/版本上下文 |
| class_id | TEXT | NOT NULL，主键组成 | 群体范围，`global` 表示课程全局 |
| version_no | INTEGER | DEFAULT 1 | 当前课程孪生版本号 |
| graph_status | TEXT |  | 图谱状态：`ready/partial/pending` |
| graph_coverage_rate | REAL | DEFAULT 0 | 知识图谱覆盖率 |
| resource_coverage_rate | REAL | DEFAULT 0 | 资源覆盖率 |
| rubric_coverage_rate | REAL | DEFAULT 0 | 评价标准覆盖率 |
| overall_mastery | REAL | DEFAULT 0 | 整体掌握度 |
| risk_level | TEXT | 索引 `idx_course_twin_profiles_risk` | 课程风险等级 |
| risk_summary_json | TEXT | NOT NULL | 风险摘要 JSON |
| payload_json | TEXT | NOT NULL | 课程孪生完整快照 |
| last_updated | TEXT |  | 业务更新时间 |
| updated_at | TEXT | NOT NULL | 记录更新时间 |

主键：`PRIMARY KEY (course_id, term_id, class_id)`

### 2. course_twin_nodes（课程孪生节点明细）

| 字段 | 类型 | 约束/索引 | 含义 |
| --- | --- | --- | --- |
| course_id | TEXT | NOT NULL，主键组成，索引 `idx_course_twin_nodes_course` | 课程 ID |
| term_id | TEXT | NOT NULL，主键组成 | 学期/版本上下文 |
| class_id | TEXT | NOT NULL，主键组成 | 群体范围 |
| node_id | TEXT | NOT NULL，主键组成 | 节点 ID |
| node_name | TEXT |  | 节点名称 |
| node_type | TEXT | 索引 `idx_course_twin_nodes_type` | 节点类型：`chapter/section/knowledge_point` |
| node_path_json | TEXT |  | 节点路径 JSON |
| prerequisite_node_ids_json | TEXT |  | 先修节点列表 |
| relation_summary_json | TEXT |  | 关系摘要 |
| resource_count | INTEGER | DEFAULT 0 | 关联资源数 |
| avg_mastery | REAL | DEFAULT 0 | 班级平均掌握度 |
| difficulty_score | REAL | DEFAULT 0 | 节点难度评分 |
| importance_weight | REAL | DEFAULT 0 | 节点重要度 |
| risk_student_count | INTEGER | DEFAULT 0 | 风险学生数 |
| recommended_action | TEXT |  | 建议动作 |
| payload_json | TEXT | NOT NULL | 节点完整快照 |
| updated_at | TEXT | NOT NULL | 记录更新时间 |

主键：`PRIMARY KEY (course_id, term_id, class_id, node_id)`

### 3. course_node_relations（课程节点关系表）

| 字段 | 类型 | 约束/索引 | 含义 |
| --- | --- | --- | --- |
| course_id | TEXT | NOT NULL，主键组成，索引 `idx_course_node_relations_course` | 课程 ID |
| from_node_id | TEXT | NOT NULL，主键组成 | 起始节点 |

| relation_weight | REAL | DEFAULT 0 | 关系权重 |

### 4. course_resource_links（课程节点-资源关联表）

> 当前 `resources` 表保存资源实体，`course_resource_links` 负责保存“资源与节点如何关联”的关系属性。

| 字段 | 类型 | 约束/索引 | 含义 |
| --- | --- | --- | --- |
| course_id | TEXT | NOT NULL，主键组成，索引 `idx_course_resource_links_course_node` | 课程 ID |

| resource_id | INTEGER | NOT NULL，主键组成，索引 `idx_course_resource_links_resource_id` | 对应 `resources.resource_id` |
| resource_role | TEXT |  | 资源角色：`core/remedial/advanced/practice` |
| match_score | REAL | DEFAULT 0 | 节点-资源匹配总分 |
| difficulty_level | TEXT |  | 资源难度：`low/medium/high` |
| matched_points_json | TEXT |  | 命中的知识点列表 |
| matched_queries_json | TEXT |  | 触发命中的检索词列表 |
| recommend_reason | TEXT |  | 推荐理由 |
| payload_json | TEXT | NOT NULL | 资源关联扩展信息 JSON |
| updated_at | TEXT | NOT NULL | 记录更新时间 |

主键：`PRIMARY KEY (course_id, node_id, resource_id)`

| source | TEXT |  | 来源：`industry/obe/manual` |
| payload_json | TEXT | NOT NULL | 映射扩展信息 JSON |

主键：`PRIMARY KEY (course_id, capability_code, node_id)`

### 6. course_evaluation_rubrics（课程评价标准表）

> 用于承接项目构想中能力维度、过程维度、反思维度等数字化评价编码。

| 字段 | 类型 | 约束/索引 | 含义 |
| --- | --- | --- | --- |
| course_id | TEXT | NOT NULL，主键组成，索引 `idx_course_rubrics_course` | 课程 ID |
| rubric_code | TEXT | NOT NULL，主键组成 | 指标编码，如 `C-A1`、`P-E2` |
| dimension_type | TEXT | 索引 `idx_course_rubrics_dimension` | 维度类型：`ability/process/reflection` |
| dimension_name | TEXT |  | 维度名称 |

# 五、当前假设与待确认

## 5.1 当前假设

- 当前课程数字孪生以 `course_id + term_id + class_id` 作为最小上下文键
- 若系统暂时没有正式 `class_id`，允许先用 `teacher_user_id + course_id` 作为代理键
- 当前资源层先以外部资源链接和现有匹配结果为主，不强依赖“自动生成资源”
- 当前图谱关系最少支持 `contains`、`prerequisite`、`related`
- 当前班级热力图、整体掌握度主要由学生数字孪生聚合得到，而不是课程侧单独计算原始明细

## 5.2 待确认项

1. 平台当前是否已经有稳定的 `class_id` / `course_instance_id`
2. 代码实践作业结果是否会有统一结构化输出，还是只提供总评结果
3. 课程数字孪生本期是否需要纳入“生成资源”能力，还是仅做“关联已有资源”
4. 评价标准编码是否由你们统一维护，还是需要课程数字孪生自行管理
5. 行业能力需求是作为静态配置导入，还是后续要支持动态更新


# 六、推荐落地顺序

## 第一阶段：先把你当前已完成的能力产品化

- 固化知识图谱写入接口
- 新增 `course_node_relations`
- 固化资源匹配写入接口
- 新增 `course_resource_links`
- 提供 `course-profile` 与 `course-node-profile` 查询接口

## 第二阶段：补足“课程孪生”而不是“课程图谱”

- 聚合学生数字孪生，形成 `course_twin_profiles` 与 `course_twin_nodes`
- 输出班级热力图、风险节点、教学重点建议
- 向路径推荐模块与诊断智能体输出结构化字段

## 第三阶段：补足课程自进化能力

- 接入教师反馈
- 接入评价标准编码
- 增加 `course_twin_history`
- 支持版本快照、课程演化与跨学期对比
