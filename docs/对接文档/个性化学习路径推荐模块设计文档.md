
### 一、现有系统构成梳理（基于文档）

当前学生数字孪生系统的核心框架：
1. **核心能力**：学生课程画像生成、画像查询、跨孪生（课程/教师）协同输出、HTTP/函数双调用模式、标准化数据更新回写机制
2. **数据底座**：包含用户主数据、学生孪生数据（主表/节点明细/历史快照）、课程数据（主表/节点）、学习行为数据（会话/LLM日志/学习计划）、评测数据（测验记录）共13张表

### 二、个性化学习路径推荐模块功能设计
完全对齐现有系统的设计规范与协同逻辑，核心功能如下：
#### 2.1 核心业务能力
##### （1）多场景路径生成
支持不同触发场景的个性化路径生成，输入输出与现有学生孪生能力匹配：
- **触发场景**：新课初始化生成、节点完成后更新、测验后调整、学生手动触发（如考前冲刺、能力提升自定义目标）
- **输入**：学生课程画像（整体掌握度、节点掌握数据、能力分层、学习习惯、风险告警）、课程结构与节点依赖、课程达标要求、自定义参数（可选）
- **输出**：分阶段的节点序列、每个节点的学习建议、预估时长、达标要求

##### （2）动态调整与版本管理
- **自动调整**：当学生学习数据异常（掌握度不达标、进度滞后、投入不足等），自动插入补学节点、跳过已掌握节点、调整后续节点难度
- **手动调整**：支持教师添加/移除节点、调整节点顺序、设置必学要求；支持学生自主跳过非核心节点、添加拓展节点
- **版本回溯**：每次调整生成新的路径版本，全量保留历史版本可审计

##### （3）进度追踪与告警
- 支持学生/教师查询当前生效路径、历史版本、完成进度
- 实时比对实际学习数据与预期要求，生成进度滞后、掌握度未达标的告警，同步到学生孪生的`risk_alerts`字段

##### （4）效果反馈与算法优化
定期统计路径完成率、学生掌握度提升率、路径匹配度（学生按路径学习的比例），反馈给推荐模型持续优化准确性。

---

#### 2.2 调用模式（与现有系统完全对齐）
##### （1）HTTP模式（前后端/跨服务）
| 接口路径 | 方法 | 入参 | 出参 |
|---------|------|------|------|
| `/api/digital-twin/student-learning-path` | POST | `student_id`(必填)、`course_id`(必填)、`force_refresh`(可选，是否强制重新生成)、`custom_params`(可选，自定义目标如截止时间、目标掌握度) | 当前生效路径完整信息（版本、节点序列、学习建议、进度、告警） |
| `/api/digital-twin/adjust-learning-path` | POST | `path_id`(必填)、`adjustment_content`(必填，调整内容JSON)、`operator_id`(必填)、`operator_type`(必填，`teacher`/`student`) | 调整后的新版本路径信息 |

##### （2）函数模式（后端内部调用）
```python
# 查询/生成学习路径
DigitalTwinModule.learning_path_service.get_learning_path(
    student_id: int, 
    course_id: str, 
    force_refresh: bool = False, 
    custom_params: dict = None
) -> dict

# 调整学习路径
DigitalTwinModule.learning_path_service.adjust_learning_path(
    path_id: int, 
    adjustment_content: dict, 
    operator_id: int, 
    operator_type: str
) -> dict
```
返回结果为Python字典，可直接序列化为JSON，符合现有系统规范。

---

#### 2.3 跨模块协同输出（符合现有1.3规范）
| 输出对象 | 输出字段                       | 输出内容                                |
| ---- | -------------------------- | ----------------------------------- |
| 学生孪生 | `outputs.for_student_twin` | 当前路径进度、下一个推荐节点、学习建议、路径相关风险告警        |
| 教师孪生 | `outputs.for_teacher_twin` | 所带学生路径完成率、共性薄弱节点、需要干预的学生列表、路径调整建议   |
| 课程孪生 | `outputs.for_course_twin`  | 高频补学节点列表、路径平均完成时长、学生节点掌握分布、课程结构优化建议 |

---

#### 2.4 数据更新与回写（符合现有1.5规范）
- 节点完成/测验提交后，实时更新路径节点状态、实际掌握度、学习时长，触发路径评估与调整
- 路径调整后写入调整日志，生成新版本并标记为当前生效
- 每日定时批量更新所有在途路径的进度状态，同步告警到学生孪生
- 所有数据优先存储在实体表，保证可查询、可审计、可扩展

### 三、配套数据库表设计
共新增3张核心表，完全兼容现有表的命名规范、字段规则（保留`username`兼容历史逻辑、`user_id`作为主关联键、`updated_at`/`payload_json`等通用字段）：
#### 3.1 `learning_path_main`（学习路径主表）
存储每个学生每门课的路径聚合信息，保留历史版本

| 字段 | 类型 | 约束/索引 | 含义 |
|------|------|----------|------|
| `path_id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 路径唯一ID |
| `user_id` | INTEGER | NOT NULL，联合索引 `idx_path_user_course`(user_id, course_id) | 学生user_id，关联`users`表 |
| `student_username` | TEXT | NOT NULL | 学生用户名，兼容历史业务 |
| `course_id` | TEXT | NOT NULL，联合索引 `idx_path_user_course` | 课程ID，关联`courses`表 |
| `path_version` | INTEGER | NOT NULL | 路径版本号，每次调整+1 |
| `is_current` | INTEGER | NOT NULL DEFAULT 0，索引 `idx_path_current` | 是否为当前生效版本（0/1），同一学生同一课程仅1条生效记录 |
| `generated_type` | TEXT | NOT NULL | 生成类型：`auto`(系统自动)/`teacher_adjusted`(教师调整)/`student_defined`(学生自定义) |
| `target_overall_mastery` | REAL | NOT NULL | 路径要求的最终整体掌握度 |
| `estimated_total_duration` | REAL | DEFAULT 0 | 预估总学习时长（分钟） |
| `start_node_id` | TEXT | NOT NULL | 路径起始节点ID，关联`course_nodes`表 |
| `end_node_id` | TEXT | NOT NULL | 路径目标节点ID，关联`course_nodes`表 |
| `status` | TEXT | NOT NULL DEFAULT `not_started` | 路径状态：`not_started`/`in_progress`/`completed`/`abandoned` |
| `progress` | REAL | DEFAULT 0 | 路径完成进度（0-1） |
| `related_plan_filename` | TEXT | | 关联现有`learning_plans`表的`filename`，兼容学习计划模块 |
| `payload_json` | TEXT | NOT NULL | 扩展信息：推荐算法参数、路径生成原因、告警信息等JSON |
| `created_at` | TEXT | NOT NULL | 路径创建时间（ISO字符串） |
| `updated_at` | TEXT | NOT NULL | 记录更新时间（ISO字符串） |

---

#### 3.2 `learning_path_nodes`（路径节点明细表）
存储每条路径的节点序列、要求与完成状态

| 字段 | 类型 | 约束/索引 | 含义 |
|------|------|----------|------|
| `path_id` | INTEGER | NOT NULL，主键组成，索引 `idx_path_nodes_path`(path_id) | 关联`learning_path_main`的`path_id` |
| `node_sequence` | INTEGER | NOT NULL，主键组成 | 节点在路径中的顺序（从1开始递增） |
| `node_id` | TEXT | NOT NULL，索引 `idx_path_nodes_node`(node_id) | 课程节点ID，关联`course_nodes`表 |
| `node_name` | TEXT | NOT NULL | 节点名称，冗余避免联表查询 |
| `node_type` | TEXT | NOT NULL DEFAULT `mandatory` | 节点类型：`mandatory`(必学)/`optional`(选学)/`remedial`(补学)/`advanced`(拓展) |
| `prerequisite_node_sequences` | TEXT | | 前置节点序列JSON数组，需完成所有前置才可学习当前节点 |
| `required_mastery` | REAL | NOT NULL DEFAULT 0.6 | 节点要求的最低掌握度 |
| `estimated_duration` | REAL | DEFAULT 0 | 预估该节点学习时长（分钟） |
| `suggested_resource_ids` | TEXT | | 推荐的资料ID列表JSON，关联`resources`表 |
| `suggested_quiz_tags` | TEXT | | 推荐的测验知识点标签JSON，匹配测验内容 |
| `learning_guidance` | TEXT | | 个性化学习建议 |
| `node_status` | TEXT | NOT NULL DEFAULT `pending` | 节点状态：`pending`/`in_progress`/`completed`/`skipped`/`failed` |
| `actual_mastery` | REAL | DEFAULT 0 | 实际掌握度，同步` twin_profile_nodes`的`mastery_score` |
| `actual_duration` | REAL | DEFAULT 0 | 实际学习时长，同步` twin_profile_nodes`的`study_duration_minutes` |
| `completed_at` | TEXT | | 节点完成时间（ISO字符串） |
| `payload_json` | TEXT | NOT NULL | 扩展信息JSON |
| `updated_at` | TEXT | NOT NULL | 记录更新时间（ISO字符串） |
> 主键：`PRIMARY KEY (path_id, node_sequence)`

---

#### 3.3 `learning_path_adjustment_logs`（路径调整日志表）
存储所有路径调整记录，用于审计、回溯与算法优化

| 字段                   | 类型      | 约束/索引                                      | 含义                                                                            |
| -------------------- | ------- | ------------------------------------------ | ----------------------------------------------------------------------------- |
| `adjustment_id`      | INTEGER | PRIMARY KEY AUTOINCREMENT                  | 调整记录唯一ID                                                                      |
| `path_id`            | INTEGER | NOT NULL，索引 `idx_adjustment_path`(path_id) | 关联调整的路径ID                                                                     |
| `adjustment_type`    | TEXT    | NOT NULL                                   | 调整类型：`auto_trigger`(系统自动)/`teacher_operation`(教师操作)/`student_operation`(学生操作) |
| `trigger_reason`     | TEXT    | NOT NULL                                   | 触发原因，如“节点X掌握度仅0.3，插入补学节点”                                                     |
| `adjustment_summary` | TEXT    | NOT NULL                                   | 调整内容摘要                                                                        |
| `old_path_version`   | INTEGER | NOT NULL                                   | 调整前的路径版本号                                                                     |
| `new_path_version`   | INTEGER | NOT NULL                                   | 调整后的路径版本号                                                                     |
| `operator_id`        | INTEGER | NOT NULL                                   | 操作人ID：系统为0，教师/学生为对应用户`user_id`                                                |
| `operator_type`      | TEXT    | NOT NULL                                   | 操作人类型：`system`/`teacher`/`student`                                            |
| `payload_json`       | TEXT    | NOT NULL                                   | 调整前后的路径完整快照JSON                                                               |
| `created_at`         | TEXT    | NOT NULL                                   | 调整时间（ISO字符串）                                                                  |

### 四、需要其他模块提供的数据
| 提供模块        | 数据内容                                                                                                       | 用途                           | 交付要求                                                                     |
| ----------- | ---------------------------------------------------------------------------------------------------------- | ---------------------------- | ------------------------------------------------------------------------ |
| 课程孪生/课程数据组  | 1. 课程节点的前置依赖关系、知识点标签、难度等级<br>2. 课程达标要求（整体掌握度阈值、核心必学节点列表）<br>3. 每个节点的必学/拓展资源列表、测验考察知识点                      | 路径生成时梳理学习顺序、匹配节点难度、匹配合适的学习资料 | 1. 在`course_nodes`/`resources`表的`payload_json`中补充标准化字段<br>2. 课程变更时主动推送通知 |
| 学生孪生模块      | 1. 实时更新的学生课程画像（整体掌握度、节点掌握度、能力分层、风险告警、学习趋势）<br>2. 学生学习行为特征（日均学习时长、资源偏好、历史学习数据）<br>3. 学生代码实操能力指标（由评测模块同步后提供） | 作为路径生成与调整的核心依据，优化路径匹配度       | 1. 提供API查询接口<br>2. 画像更新时通过事件总线推送变更通知                                     |

> 补充说明：生成的个性化路径可同步写入现有`learning_plans`表（`category`设为`personalized_path`），无需改造现有学习计划模块即可直接查询。
