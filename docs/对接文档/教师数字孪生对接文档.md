# 教师数字孪生与作业模块对接文档

# 一、需要实现哪些功能



1. 教师数字孪生（六维图）
2. 作业发布与 AI 辅助批改

目标是保证数据链路闭环

---

## 1.1 教师数字孪生（六维图）功能

### A. 当前已具备（代码已实现）

1. 教师六维画像聚合接口
   - `GET /api/dashboard/teacher-twin`
   - 返回：`overall_score`、`radar`、`dimensions`、`data_diagnosis`、`missing_data_hooks`
2. 六个维度评分计算（TeacherTwinService）
   - 专业投入 `professional_engagement`
   - 数字资源 `digital_resources`
   - 教学与学习 `teaching_learning`
   - 评估 `assessment`
   - 赋能学习者 `empowering_learners`
   - 促进学习者数字能力 `facilitating_digital_competence`
3. 数据完整度诊断
   - 输出外部指标覆盖率：`external_coverage_ratio`
   - 输出缺失指标列表：`external_metrics_missing`
4. 外部指标对接入口（已预留）
   - `POST /api/digital-twin/teacher-profile/{teacher_username}/external-sync`
   - 写入 `user_states`，key 为 `teacher_ext::{teacher_username}`
5. AI 建议按钮触发（非自动）
   - `POST /api/dashboard/teacher-twin/ai-suggestions`


### B. 还需要落地（必须做，才能避免假数据）

1. 教师互动/论坛行为真实入库
   - 至少要有：发帖、发公告、发起讨论、教师回复、学生提问、回复耗时
   - 否则 `teaching_learning` 维度会长期偏空
2. 教研协同行为真实入库
   - 至少要有：教研发帖数、共享课件数、集体备课次数
   - 否则 `professional_engagement` 维度只能靠人工灌数
3. 作业批改过程指标回流到教师孪生
   - 主观题平均批改耗时
   - 干预动作次数（布置补救任务/补救公告）
   - 风险学生干预覆盖率
4. 学生孪生预警联动
   - 将“高风险学生 + 干预记录”形成可追踪闭环
   - 否则 `empowering_learners` 维度不可信

---

## 1.2 作业发布与 AI 辅助批改功能

### A. 当前已具备（代码已实现）

1. 作业全生命周期管理
   - 新建：`POST /api/homework/assignments`
   - 列表：`GET /api/homework/assignments`
   - 详情：`GET /api/homework/assignments/{assignment_id}`
   - 修改：`PUT /api/homework/assignments/{assignment_id}`
   - 发布/关闭/重开：`/publish`、`/close`、`/reopen`
2. 学生提交与查询
   - 提交：`POST /api/homework/assignments/{assignment_id}/submissions`
   - 我的提交：`GET /api/homework/my-submissions`
3. AI 辅助出题与草稿
   - `POST /api/homework/ai/generate-questions`
   - `POST /api/homework/ai/generate-draft`
4. AI 辅助批改 + 教师终审
   - AI 批改：`POST /api/homework/submissions/{submission_id}/ai-grade`
   - 终审：`POST /api/homework/submissions/{submission_id}/final-grade`
5. 代码题自动判题（当前方案）
   - 通过 `HomeworkModule/code_judge.py` 本地沙箱执行 Python
   - 根据测试用例计算通过率并回写成绩

### B. 建议新增

1. 判题引擎抽象层（先不绑死 Judge0）
   - 定义统一接口：`judge(submission, test_cases, language)`
   - 实现两个适配器：
     - `LocalPythonJudge`（当前默认）
     - `Judge0Adapter`（后续接入，无需改前端）
2. 代码题提交前端保持现有美观页面
   - 前端继续使用现有 `frontend-vue` 作业页面
   - Judge0 仅作为后端判题服务，不暴露其原生前端
3. 作业事件写入教师孪生数据源
   - 发布作业、批改、补救任务、反馈时效，都写事件表或日志
   - 直接服务教师六维中的评估与教学维度

# 二、数据库表设计


## 2.1 已存在表（当前代码已在使用）

### 1) 用户与关系

1. `users`
   - 用户主数据（student/teacher/admin）
2. `teacher_student_links`
   - 师生绑定关系

### 2) 教师孪生基础来源

1. `sessions`
   - 教师在线时长、登录频次
2. `llm_logs`
   - 功能使用、AI 推荐/执行、部分互动与任务元数据
3. `learning_plans`
   - 资源格式、迭代次数、资源数量
4. `twin_profiles`
   - 学生掌握度（教师维度里用于班级侧推断）
5. `user_states`
   - 存放 `teacher_ext::{teacher_username}` 外部指标

### 3) 作业模块表（当前已落地）

1. `homework_assignments`
   - 作业定义
   - 关键字段：
     - `id`
     - `title`
     - `assignment_type`（subjective/objective/choice/code）
     - `class_name`
     - `due_at`
     - `allow_late`
     - `total_score`
     - `questions_json`
     - `created_by`
     - `status`
2. `homework_submissions`
   - 学生提交与评分
   - 关键字段：
     - `id`
     - `assignment_id`
     - `student_username`
     - `answers_json`
     - `status`
     - `ai_score`
     - `ai_feedback`
     - `teacher_score`
     - `teacher_comment`
     - `graded_at`
     - `grader_username`

## 2.2 需要新增表（用于替代假数据）

### 1) 教学互动事件表 `teaching_interaction_events`

用途：承载论坛/讨论/公告/回复等行为，支撑六维中的“教学与学习”。

建议字段：

1. `id` INTEGER PK AUTOINCREMENT
2. `teacher_username` TEXT NOT NULL
3. `course_id` TEXT
4. `class_name` TEXT
5. `event_type` TEXT NOT NULL
   - 枚举建议：`publish_announcement`、`start_discussion`、`teacher_reply`、`student_question`、`student_reply`
6. `target_id` TEXT
   - 讨论串/帖子/作业等业务对象 ID
7. `student_username` TEXT
8. `response_minutes` REAL
9. `payload_json` TEXT NOT NULL
10. `created_at` TEXT NOT NULL

索引建议：

1. `idx_tie_teacher_time (teacher_username, created_at)`
2. `idx_tie_event_type_time (event_type, created_at)`
3. `idx_tie_target (target_id)`

### 2) 教研协同事件表 `teaching_research_events`

用途：承载教研发帖、共享课件、集备等协同行为，支撑“专业投入”。

建议字段：

1. `id` INTEGER PK AUTOINCREMENT
2. `teacher_username` TEXT NOT NULL
3. `event_type` TEXT NOT NULL
   - 枚举建议：`research_post`、`shared_courseware`、`co_preparation`
4. `resource_id` TEXT
5. `payload_json` TEXT NOT NULL
6. `created_at` TEXT NOT NULL

索引建议：

1. `idx_tre_teacher_time (teacher_username, created_at)`
2. `idx_tre_event_type_time (event_type, created_at)`

### 3) 作业评估明细事件表 `homework_grading_events`

用途：把“批改耗时、补救动作、是否 AI 执行”结构化，支撑“评估”“赋能学习者”。

建议字段：

1. `id` INTEGER PK AUTOINCREMENT
2. `assignment_id` TEXT NOT NULL
3. `submission_id` TEXT
4. `teacher_username` TEXT NOT NULL
5. `student_username` TEXT
6. `event_type` TEXT NOT NULL
   - 枚举建议：`ai_grade`、`teacher_final_grade`、`remediation_material`、`remediation_announcement`
7. `grading_minutes` REAL
8. `is_ai_recommended` INTEGER DEFAULT 0
9. `is_ai_executed` INTEGER DEFAULT 0
10. `payload_json` TEXT NOT NULL
11. `created_at` TEXT NOT NULL

索引建议：

1. `idx_hge_teacher_time (teacher_username, created_at)`
2. `idx_hge_assignment (assignment_id)`
3. `idx_hge_submission (submission_id)`

# 三、需要别人提供的数据和模块


## 3.1 必须提供（否则六维图仍会失真）

### A. 论坛/互动模块（当前缺口最大）


1. 必须提供：
   - 教师公告次数
   - 教师发起讨论次数
   - 教师回复率
   - 平均响应时长
2. 接入方式（二选一）：
   - 方式1：写 `teaching_interaction_events`
   - 方式2：按天汇总后调用 `external-sync`
3. 影响维度：`teaching_learning`

### B. 教研协同模块

1. 必须提供：
   - 教研发帖数 `research_posts`
   - 共享课件数 `shared_courseware`
   - 集体备课次数 `co_preparation_count`
2. 接入方式：`teaching_research_events` 或 `external-sync`
3. 影响维度：`professional_engagement`

### C. 作业批改过程数据

1. 必须提供：
   - 主观题平均批改耗时 `subjective_grading_minutes`
   - AI 推荐动作执行率
   - 补救动作次数（补救任务/补救公告）
2. 接入方式：`homework_grading_events`（推荐）或写入 `llm_logs.metadata`
3. 影响维度：`assessment`、`teaching_learning`

### D. 学生风险联动数据

1. 必须提供：
   - 风险学生名单
   - 风险等级与变化趋势
   - 干预后是否改善
2. 接入方式：教师模块消费学生孪生接口 + 干预行为回写
3. 影响维度：`empowering_learners`

---

## 3.2 建议协作接口清单

1. 论坛模块 -> 教师孪生
   - `POST /api/teacher-events/interaction`（建议新增）
   - 或每日汇总推送到 `external-sync`
2. 教研模块 -> 教师孪生
   - `POST /api/teacher-events/research`（建议新增）
3. 作业模块 -> 教师孪生
   - `POST /api/teacher-events/grading`（建议新增）
4. 教师孪生统一汇总
   - `GET /api/dashboard/teacher-twin`

---

## 3.3 数据打通验收标准


1. 六个维度都来自真实行为数据或真实汇总数据
2. 任一维度可追溯到原始记录（事件表或日志）
3. 作业批改数据可在教师孪生中体现出趋势变化
4. 学生风险干预有“触发-执行-结果”闭环链路
