# 数字孪生指标体系总表（实体优先）

> 同步日期：2026-04-14

## 1. 目标

统一学生孪生、课程维度画像、教师孪生三类指标口径，并对齐当前实体化存储。

## 2. 学生孪生

### 输入指标（节点级）
- `quiz_score`
- `progress`
- `study_duration_minutes`
- `llm_interaction_count`
- `mastery_score`

来源：`twin_profile_nodes`

### 输出指标（摘要级）
- `overall_mastery`
- `technical_level`
- `radar`
- `weak_nodes`
- `risk_alerts`
- `trend`

来源：`twin_profiles + twin_profile_nodes + twin_history`

## 3. 课程维度画像

接口：`POST /api/digital-twin/student-course-profile`

关键输出：
- `course_node_count`
- `matched_nodes[]`（`node_id/mastery_score/progress/quiz_score`）
- `node_summary`（节点数、弱项数、平均进度、平均测验分）

课程实体基础：
- `courses`
- `course_nodes`
- `resources`

## 4. 测试维度（新增实体）

表：`quiz_attempts`

核心字段：
- `attempt_id`
- `user_id`
- `course_id`
- `node_id`
- `score/total/passed`
- `created_at`

用途：
- 追踪学生测试轨迹
- 支撑课程/节点测验分析
- 为孪生趋势与风险提供时间序列证据

## 5. 教师孪生

关键输出：
- `overall_score`
- `dimensions`
- `student_scope`
- `data_diagnosis`

依赖：
- `teacher_student_links`
- 学生孪生聚合结果
- 教师外部同步指标（`external-sync` 写入）

## 6. 指标口径建议

1. 核心评估：学术表现 + 学习行为（高置信）
2. 扩展评估：能力迁移 + 职业匹配（中置信）
3. 观察指标：社会情感（低置信，不直接计入总分）

## 7. 当前实现特征

- 课程/资源/测试已实体化。
- 课程相关接口读写已实体优先。
- 孪生输出保持兼容，但底层数据来源已迁移到更结构化的数据表。
